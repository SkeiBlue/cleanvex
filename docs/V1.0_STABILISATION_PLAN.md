## Sauvegarde et restauration

L'export ZIP applicatif contient:

- `manifest.json` avec les donnees principales de l'utilisateur;
- `files/` avec les fichiers prives presents sur disque.

Pour une restauration complete de production, conserver aussi:

- dump PostgreSQL realise cote serveur;
- archive du dossier `PRIVATE_FILES_DIR`;
- version du code et migrations Prisma.

Procedure minimale:

1. Restaurer PostgreSQL depuis le dump.
2. Restaurer le dossier prive des fichiers.
3. Lancer `npm run prisma:generate --prefix backend`.
4. Lancer `npm run validate`.
5. Verifier ouverture d'un document prive depuis l'interface.
