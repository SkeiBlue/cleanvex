# V2.0 - Creation de compte avec email verifie

## Objectif

Preparer l'ouverture controlee de comptes sans rendre l'application publique libre.

## Inclus

- Inscription via `POST /api/auth/register`.
- Code d'invitation optionnel via `SIGNUP_INVITE_CODE`.
- Compte cree avec `emailVerified = false`.
- Login refuse tant que l'email n'est pas verifie.
- Token de verification email hashé en base.
- Verification via `POST /api/auth/verify-email`.
- Renvoi via `POST /api/auth/resend-verification`.
- Envoi SMTP optionnel si `SMTP_HOST` est configure.
- Logs audit et activite.
- UI minimale sur l'ecran de connexion.

## Comportement dev

En environnement non production, le token de verification est renvoye par l'API pour faciliter les tests locaux sans SMTP.

## Comportement production attendu

En production, le token n'est pas renvoye. Configurer un SMTP pour envoyer le lien:

```env
APP_PUBLIC_URL="https://ton-domaine.fr"
SMTP_HOST="smtp.example.com"
SMTP_PORT="587"
SMTP_SECURE="false"
SMTP_USER="user"
SMTP_PASS="password"
MAIL_FROM="Personal Platform <no-reply@ton-domaine.fr>"
```

`MAIL_FROM` doit etre autorise par le compte SMTP. Par exemple, si `SMTP_USER=contact@domaine.fr`, beaucoup de fournisseurs refusent un expediteur `no-reply@domaine.fr`.

## Non inclus

- Gestion admin des invitations.
- Reset mot de passe.
- Limitation avancee anti-spam par email.
